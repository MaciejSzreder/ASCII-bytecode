Literals

In order to load a constant value into the accumulator register, use the _ clear instruction and digits representing the value. You can use decimal digits 0123456789, or binary digits OI (uppercase O and uppercase I). E.g. _123 stores 123 in the accumulator register, and _IOI stores 5 in the accumulator register.

Code formatting

In order to allow for better code readability, no operation instructions are whitespace characters: space, horizontal tabulation, line feed, carriage return.

Registers assignment

Copying values happens between a register and the accumulator register, an exception is the stack register. To copy a register's value to the accumulator register, use a lowercase instruction being a single-letter name of the register. To copy the accumulator's value to a register, use an uppercase instruction being a single-letter name of the register. Assigning a value to the stack register is only possible by swapping its value with the data address register's value by the ( stack swap instruction.

Binary Operators

Binary operators operate on two registers: the argument register and the accumulator register. The result is stored in the accumulator register. For most binary operator instructions, the left operand is the accumulator register, and the right operand is the argument register. An exception is the \ inverted division instruction, where the operator is traditional division and operands are swapped.

Binary operator instructions:

  • + - addition
  • - - subtraction
  • * - multiplication
  • / - division
  • \ - inverted division
  • % - modulo
  • & - bitwise AND
  • | - bitwise OR
  • ^ - bitwise XOR
  • < - less
  • = - equal
  • > - greater
  • ? - random
  • M - maximum
  • m - minimum

+, -, * wraps around if the result overflows a 16-bit two's complement register. E.g. 32767+1=-32768, 0-(-32768)=-32768, (-1)*(-32768)=-32768.

If the divisor is 0 for %, the result is 0.

If the divisor is 0 and the dividend is positive (in / or \), the result is the maximal positive value.

If the divisor is 0 and the dividend is negative (in / or \), the result is the minimal negative value.

If the divisor and the dividend are 0 (in / or \), the result is 0.

Unary Operators

Unary operators operate only on the accumulator register. It is the operand and the result is stored in it.

Unary operator instructions:

  • ! - logical NOT
  • ~ - bitwise NOT
  • # - popcount
  • : - byte swap
  • s - sine
  • S - sign extension

Control Flow

Control flow instructions change the default execution process by changing the instruction pointer register or changing the execution mode.

  • ␀ main loop instruction (non-printable character ending the code) - sets the instruction pointer to 0, so the program executes from the beginning. It makes the program work in a main loop.
  • ; ignore instruction - unconditionally skips code execution until the next ; ignore instruction. Useful for commenting code.
  • [ conditional ignore instruction - skips code until the corresponding ] end conditional instruction, if the accumulator value is 0. The ] end conditional instruction between ; ignore instructions is ignored. Each [ conditional ignore instruction must have a corresponding ] end conditional instruction in sequential order, and the contained code must jump back (if jumps) before the closing ] end conditional instruction, otherwise the behavior is undefined. Useful for an if statement.
  • ] end conditional instruction - marks the end of skipped code by [ conditional ignore instruction. Useful for an if statement.
  • } conditional repeat - repeats code from the corresponding { repeat begin instruction until the accumulator value is 0. The { repeat begin instruction inside ; ignore instruction is ignored. Each } conditional repeat instruction must have a corresponding { begin repeat instruction in sequential order, and contained code must jump back (if jumps) after opening { begin repeat instruction, otherwise the behavior is undefined.
  • { begin repeat instruction - marks the beginning of code repeated by the } conditional repeat instruction. Useful for loops.
  • J jump instruction - Jumps to the instruction pointed to by the accumulator register by copying its value to the instruction pointer register.
  • j read J instruction - does not modify execution flow, but can be used to control flow by copying the instruction pointer value into the accumulator register, modify it, and copy back to the instruction pointer register. Useful for relative jumps.
  • ) call instruction - pushes the return address onto the stack pointed to by the data address register, then jumps to the instruction pointed to by the accumulator register. Useful for function calls.
  • Y fork instruction - creates a new hardware thread by copying the current core state to a new hardware context (e.g. a free hardware core, a free logic core, a new entry in the register file), but the instruction pointer register of the new hardware thread is set to the accumulator value. Useful for code forking.
  • y exit instruction - deletes the current hardware thread.

Memory

Memory is organized as a linear array of bytes. Instructions operating on memory use 2 registers: the data address register and the accumulator register. The data address register holds a byte address. The accumulator register holds a value to write or a loaded value. Memory instructions:

  • @ load instruction
  • $ store instruction
  • , push instruction
  • ` pop instruction

Input/Output

Input and output instructions use 2 registers: the port register and the accumulator register. The port register holds an index of the input/output register, and the accumulator register holds a value to write or a loaded value. A single port can store a single byte of data. Input/output instructions:

  • i read input instruction
  • o write output instruction

Screen

There is only one instruction to manipulate the screen: . set pixel instruction. It uses 5 registers: X, Z, R, G, B. Registers X and Z hold coordinates of the pixel, and registers R, G, B store the new color of the pixel. Register X stores the x coordinate, register Z stores the y coordinate, the lower byte of register R stores the red channel, the lower byte of register G stores the green channel, the lower byte of register B stores the blue channel. Color is encoded in TrueColor format.

Execution modes

There are at least two execution modes:

  • Execution mode - fetch instructions pointed to by the instruction pointer register
  • Service mode - fetch instructions from the service port

A device starts in service mode, and switches to execution mode after receiving the ␀ main loop instruction.

Note that service mode changes the effects of some instructions due to lack of flow control. E.g.:

  • ␀ main loop instruction - enters execution mode
  • J jump instruction - will not jump
  • } conditional repeat instruction - will not repeat code
  • ) call instruction - will not jump

Memory

Memory is an array of 8-bit (1 byte) wide cells. Each cell has an address, if the memory has fewer cells than possible addressable space, some cells can have multiple addresses. The addressable space is determined by the Data Address register and the Instruction Pointer register, so there are 216=65536 addresses. The first cell has address 0, the second 1, the last -1, and the penultimate -2.

Instructions operating on memory

  • @ load instruction
  • $ store instruction
  • , push instruction
  • ` pop instruction
  • ) call instruction (pushes return address)
