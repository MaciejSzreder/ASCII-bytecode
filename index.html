Literals

In order to load a constant value into accumulator register, use the _ and digits presenting the value. You can use decimal digits 0123456789, or binary digits OI (big letter O and big letter I). Eg. _123 stores 123 in the accumulator register, and _IOI stores 5 in the accumulator register.

Code formatting

In order to allow for better code readability no operation instructions are whitespaces: space, horizontal tabulation, line feed, carriage return.

Registers assignment

Coping values happens between the register and the accumulator register, exception is stack register. To copy register value to accumulator register use small letter instruction being single letter name of the register. To copy accumulator value to register use capital letter instruction being single letter name of the register. To assign value to stack register is possible only by swapping its value with data address value by (.

Binary Operators

Binary operators operate on two registers: argument register and accumulator register. Result is stored in the accumulator register. For most binary operator instruction left operand is the accumulator register and right operand is the argument register. Exception is \ (inverted division), where the operator is traditional division and operands are swapped.

Binary operator instructions:

  • + - addition
  • - - subtraction
  • * - multiplication
  • / - division
  • \ - inverted division
  • % - modulo
  • & - bitwise AND
  • | - bitwise OR
  • ^ - bitwise XOR
  • < - less than
  • = - equals
  • > - greater than
  • ? - random
  • M - max
  • m - min

Unary Operators

Unary operators operate only on the accumulator register, it is the operand and the result is stored in it.

Unary operator instructions:

  • ! - logical NOT
  • ~ - bitwise NOT
  • # - popcount
  • : - byte swap
  • s - sinus
  • S - sign extension

Control Flow

Control flow instructions changes the default execution process by change instruction pointer register or changing execution mode.

  • ␀ (not printable character ending the code) - sets the instruction pointer to 0, so the program executes from beginning. It makes the program works in main loop.
  • ; - unconditional skip code until next ;. Useful for commenting code.
  • [ - skips code until corresponding ], if accumulator value is 0. ] inside ; are ignored. Each [ have to have corresponding ] in sequential order, and contained code have to jump back (if jumps) before closing ], otherwise the behavior is undefined. Useful for if statement.
  • ] - marks end of skipped code by [. Useful for if statement.
  • } - repeats code since corresponding { until accumulator value is 0. { inside ; are ignored. Each } have to have corresponding { in sequential order, and contained code have to jump back (if jumps) after opening {, otherwise the behavior is undefined.
  • { - marks beginning of code repeated by }. Useful for loops.
  • J - Jumps to instruction pointed by accumulator register by coping its value to instruction pointer register.
  • j - do not modify execution flow, but can used to control flow by coping instruction pointer value into accumulator register, modify it and copy back to instruction pointer register. Useful for relative jumps.
  • ) - push return address to stack pointed by data address register, then jumps to instruction pointed by accumulator register. Useful for function calls.
  • Y - creates new hardware thread by coping current core state to new hardware context (eg. free hardware core, free logic core, new entry in register file), but instruction pinter register of new hardware thread is set to accumulator value. Useful for code forking.
  • y - deletes current hardware thread execution.

Memory

Memory is organized as a linear array of bytes. Instruction operating on memory uses 2 registers: data address register and accumulator register. Data address register holds byte address. Accumulator register holds value to write or loaded value. Memory instructions:

  • @ - loads value from memory
  • $ - writes value to memory
  • , - pushes value to end of stack; it decrements data address register before writing
  • ` - pops value from end of stack; it increments data address register after reading

Input/Output

Input and output instructions uses 2 registers: port register and accumulator register. Port register holds index of input/output register and accumulator register holds value to write or loaded value. Single port can store single byte of data. Input/output instructions:

  • i - loads value from port
  • o - writes value into port

Screen

There is only one instruction to manipulate screen: .. It uses 5 registers: X, Z, R, G, B. Registers X and Z holds coordinates of pixel, and registers R, G, B stores new color of pixel. Register X stores x coordinate, Register Z stores y coordinate, lower byte of register R stores red channel, lower byte of register G stores green channel, lower byte of register B stores blue channel. Color is encoded in TrueColor format.

Execution modes

There are at least two execution modes:

  • Execution mode - fetch instructions pointed by instruction pointer register
  • Service mode - fetch instructions from service port

Device starts in service mode, and switches to execution mode after receiving ␀ instruction.

Mind that service mode changes effects of some instructions due lack of flow control. E.g. :

  • ␀ - enters execution mode
  • J - will not jump
  • } - will not repeat code
  • ) - will not jump
